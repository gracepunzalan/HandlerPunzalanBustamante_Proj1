Project 1 Description

According to the Machine Repository description of our dataset, "This is a transnational data set which contains all the transactions occurring between 01/12/2010 and 09/12/2011 for a UK-based and registered non-store online retail.The company mainly sells unique all-occasion gifts. Many customers of the company are wholesalers."

We plan to look at the month of December and analyze trends in where purchases were made from, at what price, etc. in order to better understand the demographic of consumers at the retail store. We will depict relationships between the attributes and allow the user to choose what they view with interactivity and access more information as they interact with our graphics.


Data Description

The name of our dataset is "Online Retail Data Set." Here is the link to the source.

There are 8 attributes. The attributes are: Invoice number , Stock code , Description, Quantity, invoice date, unit price, customer ID, and country.

Further description from the UCI website is pasted below:
InvoiceNo: Invoice number. Nominal, a 6-digit integral number uniquely assigned to each transaction. If this code starts with letter 'c', it indicates a cancellation.
StockCode: Product (item) code. Nominal, a 5-digit integral number uniquely assigned to each distinct product.
Description: Product (item) name. Nominal.
Quantity: The quantities of each product (item) per transaction. Numeric.
InvoiceDate: Invice Date and time. Numeric, the day and time when each transaction was generated.
UnitPrice: Unit price. Numeric, Product price per unit in sterling.
CustomerID: Customer number. Nominal, a 5-digit integral number uniquely assigned to each customer.
Country: Country name. Nominal, the name of the country where each customer resides

Sketch Of Interface


List of Interactions

We will have a line chart with y-axis as days of December and x-axis as either sum of unit prices or sum of quantities, depending on what the user checks. The user can only check one of these at a time. We will also have a bubble chart, where each bubble will represent that most commonly bought item for a country, and its sized will be based on what that quantity was.

For the line graph, the user can hover over a line to show which country that line represents and see the associated number for that point in the data set. The user can click a line to bold that line, then click again to unbold. Multiple lines can be clicked at one so as to make it clearer to compare. The user can also check "sum of unit prices" or "sum of quantities" (only one at a time) to change what the x-axis shows and the chart will adjust and update accordingly.

For the bubble chart, we will have a check box for top 5 most bought items. Once the user clicks on one of these items, circles representing countries will be drawn, where the radius varies depending on how many of that item was bought the given country.
